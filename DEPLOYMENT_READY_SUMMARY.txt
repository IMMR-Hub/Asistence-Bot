 ARCHITECTURE AT A GLANCE
───────────────────────────────────────────────────────────────────────────────

Customer sends WhatsApp message
         ↓
Twilio webhook receives message
         ↓
/webhooks/whatsapp/inbound endpoint (with tenant_slug)
         ↓
Message Parser (detects Twilio format)
         ↓
Message Processor (finds/creates conversation)
         ↓
Response Orchestrator
    ├── Load ConversationMemory from DB
    ├── Apply state machine logic
    ├── Route to state-specific handler
    ├── Generate response via Groq/Ollama/Template
    └── Save updated memory to DB
         ↓
Send WhatsApp response via Twilio
         ↓
Customer sees response ✅

Each message advances state machine through predetermined paths:
- URGENT cases → COLLECTING_IDENTITY → URGENT_ESCALATED_WAITING → CLOSED
- BOOKING cases → COLLECTING_PREFERENCE → OFFERING_SLOTS → WAITING_CONFIRMATION → BOOKED → CLOSED
